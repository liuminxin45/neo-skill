# skill-creator

## What this skill does
对话式收集需求（最多 10 个关键问题），生成可在 Claude/Windsurf/Cursor/GitHub Skills 等多 AI Assistant 运行的技能包（SKILL.md + references/scripts/assets），并输出 Claude .skill 打包文件。

## When to use (examples of user requests)
- 我想做一个 skill（Claude/Windsurf/Cursor 都能用）
- 帮我生成 SKILL.md
- 把我的 prompt 工作流变成 skill
- 做一个 skill 库/skill creator
- 打包成 Claude .skill

## Degrees of freedom
This skill is designed for **low** freedom (high=heuristic, low=deterministic scripts).

## Requirement collection (at most 10 questions)
Ask only what is missing. Never exceed 10 questions total.
1. 这个 skill 的一句话目标是什么（动词开头，面向可重复任务）？
2. 用户会用哪些触发语句/关键词来表述这个需求（至少 5 条）？
3. 期望输出是什么形态（报告/代码/补丁/命令/文件树/清单等）？
4. 硬约束有哪些（例如不可改变行为/不可联网/必须跑测试等）？
5. 允许使用的工具边界是什么（shell/git/python/读写文件/网络等）？
6. 需要哪些输入（repo/commit/diff/config/logs 等），从哪里来？
7. 工作流要拆成哪些关键步骤/分支？关键 gate 是什么？
8. 常见失败/边界情况有哪些？希望如何处理（停止/回退/提示）？
9. 为了减少上下文占用，哪些内容应放到 references/ 或 scripts/ 里？
10. 需要适配哪些 AI Assistant（Claude/Windsurf/Cursor/VS Code/GitHub Copilot 等）？

## Workflow
*Mode:* `sequential`

### Step 1 — 对话式收集需求（<=10 问）
*Type:* `action`
只问缺失信息；最多 10 个问题；问完就进入生成阶段。
### Step 2 — 落地 SkillSpec（单一真源）
*Type:* `action`
把收集到的信息写入 skills/<name>/skillspec.json；确定 references/scripts/assets 需要哪些内容（progressive disclosure）。
### Step 3 — 生成多 Assistant 入口文件
*Type:* `action`
输出：.claude/.windsurf/.cursor/.github 对应入口 + resources 拷贝。

```bash
python3 .shared/skill-creator/scripts/generate.py skills/<skill>/skillspec.json
```

### Step 4 — 校验（spec + Claude frontmatter strict + 目录噪声）
*Type:* `gate`
Claude strict：frontmatter 仅允许 name/description；skill bundle 内禁止 README/CHANGELOG 等无关文档。

```bash
python3 .shared/skill-creator/scripts/validate.py skills/<skill>/skillspec.json
```

### Step 5 — 打包分发（Claude .skill + 仓库 zip）
*Type:* `action`
Claude：zip 根是 skill 文件夹；仓库 zip 便于直接丢进 IDE。

```bash
python3 .shared/skill-creator/scripts/package.py --target claude --skill <skill>
python3 .shared/skill-creator/scripts/package.py --target repo
```
